## DSpark 核心机制

### 1. 半自回归草稿架构
- **并行主干**：基于 DFlash 改进，一次前向产出 block_size 个候选位置的 hidden + base_logits
  - 对 target 多层 hidden states 做 cross-attention（K/V 来自 target，Q 来自 noise embedding）
  - 主干输入：第 0 位放真实 anchor token，其余位置填 mask_token
- **顺序头**：在 base_logits 上加 bias，逐 token 注入前缀依赖
  - `VanillaMarkov`：仅依赖前一个 token，`bias = W2·W1[x_{k-1}]`
  - `GatedMarkovHead`：带门控
  - `RNNHead`：GRU 式递归状态 `s_k = g·s_{k-1} + (1-g)·tanh(...)`

### 2. 置信度调度验证
- 草稿模型每个候选位置输出置信度分数（AcceptRatePredictor = Linear(d,1)）
- 训练目标对齐：`accept_rate = 1 - 0.5·∑|p_draft - p_target|`，BCE 回归
- 部署期：硬件感知前缀调度器，低并发给 4-6 token，高并发缩减

### 3. 训练 Loss
```
L = α_ce·CE(draft, target_ids)          # α=0.1
  + α_l1·L1(softmax(draft), softmax(target))  # α=0.9
  + α_conf·BCE(confidence, accept_rate)       # α=1.0
```
位置衰减权重 `exp(-pos/γ)`，γ=4.0

### 4. 关键工程
- Anchor 采样：num_anchors=512，每锚点预测 block_size=7
- 通信优化：并行训练只传 hidden 不传 vocab logits，O(V)→O(d)
- block mask：每 block 只看自己 anchor 之前的 context + 自己 block 内 draft

## RWKV-7 适配分析

### 架构匹配度（高）
RWKV-7 cell 本质就是"逐 token 递归、把前缀压缩进固定状态、输出 bias"——与 DSpark 顺序头同构。

### RWKV-7 Delta Rule（来自 参考/web-rwkv time_mix_v7.wgsl）
```
# 每 head 的 64×64 矩阵状态 ss 更新
sa = sum_j shared_a[h+j] * state[j]   # 删除项（向量化学习率）
bb = kk * a                            # 添加项系数
ss[c] = ss[c] * w[c] + kk[c] * vv + sa * bb[c]
# w = exp(-0.606531 * sigmoid(x))  # 衰减
```
标准形式：`s = s·w + k⊗v + (a·s)⊗k`

### 关键约束（来自 project_memory）
- RWKV-7 state 是非线性递归乘积，线性状态操作无法绕过 Delta Rule 非线性
- Logit Fusion 是唯一稳定的 RWKV state 组合方案
- 单 GPU + web-rwkv 架构下 speculative decoding 曾导致 4.5× 退化（GPU 争抢）

## 复现阶段

### 阶段 1：顺序头单测（当前）
**目标**：验证 RWKV-7 cell 作为顺序头能否比 GRU head 接受率更高
**方法**：合成数据，隔离测试"顺序头"这个变量
- Target：固定小 LSTM 生成 token 序列 + logits（数据源）
- Draft 主干：直接用 target hidden（跳过并行主干训练，专注顺序头）
- 顺序头三组：None / GRU(RNNHead) / RWKV-7 Delta
- 指标：block 内各位置 top-1 接受率、CE loss

### 阶段 2：并行主干 + RWKV 顺序头（小规模）
真实 RWKV-7 0.4B 作为 target，2 层主干 + RWKV cell 顺序头，cross-attn 到 target hidden

### 阶段 3：置信度头 + 离线调度评估
加 AcceptRatePredictor，离线算理论吞吐曲线

### 阶段 4：端到端集成（高风险）
接 web-rwkv 推理路径。需阶段 3 数据支撑
