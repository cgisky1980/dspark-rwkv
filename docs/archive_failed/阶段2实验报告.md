# DSpark → RWKV 复现 · 阶段2实验报告

## 实验目标

用真实 RWKV-7 0.1B 作为 target，验证 DSpark 双轨架构（并行主干 + 顺序头）在真实 LLM 上的可行性，对比 GRU vs RWKV-7 顺序头。

## 实验配置

| 参数 | 值 |
|---|---|
| target | RWKV-7 0.1B (L=12, C=768, H=12, N=64, V=65536) |
| target hidden 采样层 | layer 0/6/11（首中尾） |
| 并行主干 | 2 层 Transformer (self-attn + cross-attn + ffn) |
| draft hidden 维度 | 256 |
| block_size | 4 |
| 训练步数 | 1500 |
| batch size | 64 |
| LR | 1e-4 |
| 训练数据 | 512 序列 × 32 token（target 自回归生成） |
| 顺序头 rank | 128 |
| Loss | CE（label=target token） |
| 设备 | RTX 2080 Ti 22GB, torch 2.6.0+cu124 |

## 架构

```
并行主干                          顺序头
├─ noise embedding               ├─ GRU (DSpark RNNHead)
│  ├─ 位置0: anchor token emb    └─ RWKV-7 DPLR (token shift + LayerNorm)
│  └─ 位置1+: mask token
├─ target hidden 融合            置信度头
│  └─ 3×768 → 256 (fc + LN)     └─ Linear(256, 1)
├─ 2× CrossAttnLayer
│  ├─ self-attn (block 内)
│  ├─ cross-attn (→ target hidden)
│  └─ ffn
├─ final LayerNorm
└─ lm_head (256 → 65536)
```

## 结果

### 表1：接受率对比

| 顺序头 | 参数量 | 平均 | 位置0 | 位置1 | 位置2 | 位置3 |
|---|---|---|---|---|---|---|
| GRU (DSSpark) | 53.2M | 0.3159 | 0.381 | 0.307 | 0.273 | 0.303 |
| **RWKV-7 DPLR** | 61.7M | **0.3271** | **0.447** | 0.312 | 0.283 | 0.266 |

### 关键发现

1. **DSpark 双轨架构在真实 RWKV-7 target 上跑通**：
   - 接受率 30%+，远高于随机基线（1/65536 ≈ 0.0015%）
   - 证明架构可行，RWKV target 的 hidden states 可被 draft 的 cross-attn 有效利用

2. **RWKV-7 顺序头整体略优于 GRU**：0.3271 vs 0.3159（+0.011）
   - 位置 0 RWKV 0.447 > GRU 0.381（**首位强 17%**）
   - 矩阵状态容量优势在首位体现：RWKV 能从 anchor hidden 提取更多信息

3. **位置 3 RWKV 0.266 < GRU 0.303**：
   - 递归状态在长 block 末位衰减
   - 但整体平均仍优于 GRU

4. **训练曲线**：
   - 两者都在 1500 步内持续收敛
   - RWKV-7 收敛稍慢但最终更高
   - loss 从 5.0 降到 3.4，acc 从 13% 升到 33%

## 与阶段1的对比

| 维度 | 阶段1（合成任务） | 阶段2（真实 RWKV target） |
|---|---|---|
| target | 无（weak hidden 直接给） | RWKV-7 0.1B |
| vocab | 256 | 65536 |
| block_size | 8 | 4 |
| GRU 接受率 | 0.83-1.00 | 0.316 |
| RWKV 接受率 | 0.80-1.00 | 0.327 |
| RWKV vs GRU | 接近/略低 | **略高** |

阶段2在真实 LLM 上 RWKV 顺序头反超 GRU，而阶段1合成任务上两者接近。可能原因：
- 真实 LLM 的 hidden states 信息密度高，矩阵状态容量优势能发挥
- 真实 vocab 大（65536），GRU 向量状态的瓶颈更明显

## 局限与改进方向

1. **接受率 30% 偏低**：
   - 训练步数少（1500），可加到 5000+
   - draft 参数量小（53-62M vs target 100M+）
   - cross-attn 只看 anchor 单位置 hidden，信息量有限

2. **cross-attn 改进**：
   - 当前 K/V = anchor 位置的单 hidden
   - 应改为 K/V = anchor 之前多个位置的 hidden（真正的 context）

3. **缺 L1 loss**：
   - 只用 CE loss
   - 加 L1 loss（需存 target logits 或实时算）可能提升

4. **block_size=4 偏小**：
   - DSpark 默认 5-7
   - 可试 block_size=6 或 8

5. **置信度头未训练**：
   - 架构里有 conf_head 但 loss 里没加 BCE
   - 需要加置信度标签（累积接受率）和 BCE loss

## 结论

**阶段2核心目标达成**：DSpark 双轨架构（并行主干 + 顺序头）在真实 RWKV-7 0.1B target 上可行，RWKV-7 顺序头在真实 LLM 任务上整体略优于 GRU，尤其首位接受率显著更高（0.447 vs 0.381）。

这修正了阶段1合成任务上"两者接近"的结论——在真实 LLM 场景下，RWKV 矩阵状态的容量优势能体现出来。
