## 关键结论

1. **公式正确性是基础**：v1 用错误公式 `(a·s)⊗k` 得出"RWKV 首位弱"的错误结论，正确 DPLR 公式 `S*w+(S@kk)*(-kk*a)+v⊗k` 后首位不再弱。

2. **token shift 是决定性因素**：双 key 任务 0.136 → 0.571（4.2 倍）。没有 token shift，Delta Rule 缺少时间混合信号，状态更新质量差。这回答了"阶段一是否用了 token shift"——之前没用，是表现差的主因。

3. **LayerNorm 在复杂任务上有额外收益**：0.571 → 0.799（+0.23）。单 key 任务无增益（已饱和），双 key 任务 LN 让训练更稳。证实 RWKV-7 用 LayerNorm 而非 RMSNorm 是对的。

4. **RWKV 矩阵状态在多 key 累积上有真实优势**：位置 2（需跨 key 累积）RWKV 0.333 > GRU 0.250。即使整体平均略低于 GRU，关键位置反超。

5. **之前"RWKV 不如 GRU"的结论是实现缺陷导致**，非架构问题。完整配置（正确公式 + token shift + LayerNorm）下 RWKV 接近 GRU，且在关键位置反超。
